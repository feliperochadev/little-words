## Summary

Fixed five iOS UX bugs: faded language buttons (hexToRgba), broken date picker in EditProfileModal (renderAsOverlay + Keyboard.dismiss), keepsake preview sizing (-10%), button font overflow (-15%), and AddWordModal keyboard scroll (delayed focus via InteractionManager).

## Design Decisions Made

1. **Bug 1 â€” hexToRgba over withOpacity:** Added `hexToRgba()` helper instead of modifying `withOpacity()` globally. Only language button backgrounds use rgba; other `withOpacity` callers unaffected.
2. **Bug 2 â€” renderAsOverlay prop:** Added `renderAsOverlay` prop to `WheelDatePickerModal` instead of always rendering as overlay. Preserves backward compat for onboarding screen (standalone Modal). EditProfileModal passes `renderAsOverlay` to avoid iOS stacked-modal issue.
3. **Bug 2 â€” Keyboard.dismiss before picker open:** Added explicit `Keyboard.dismiss()` call in date button handler to prevent Android double-tap caused by keyboard dismiss stealing first tap.
4. **Bug 5 â€” InteractionManager over setTimeout:** Used `InteractionManager.runAfterInteractions()` instead of a fixed delay for word input focus. More reliable across devices â€” waits for modal animation to complete.

## Enhancements

### 2026-04-15_6 â€” Home mini-timeline date deduplication + Android modal focus blink

- **Description:** (1) Home mini-timeline now deduplicates dates the same way as the memories screen â€” consecutive items on the same day show the date only once. (2) Android modal blink on open fixed: root cause is Android's `adjustResize` layout shift when a TextInput gains focus after the modal animation completes. Fix: auto-focus is now iOS-only (`Platform.OS !== 'android'`) in both `AddWordModal` (word input) and `AddCategoryModal` (name input). On iOS, delayed focus (350ms = `TIMING.MODAL_FOCUS_DELAY`) retained for smooth KeyboardAwareScrollView scroll. On Android, users tap to focus â€” no layout jump, no blink.
