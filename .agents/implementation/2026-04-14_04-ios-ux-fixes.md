## Summary

Fixed five iOS UX bugs: faded language buttons (hexToRgba), broken date picker in EditProfileModal (renderAsOverlay + Keyboard.dismiss), keepsake preview sizing (-10%), button font overflow (-15%), and AddWordModal keyboard scroll (delayed focus via InteractionManager).

## Design Decisions Made

1. **Bug 1 — hexToRgba over withOpacity:** Added `hexToRgba()` helper instead of modifying `withOpacity()` globally. Only language button backgrounds use rgba; other `withOpacity` callers unaffected.
2. **Bug 2 — renderAsOverlay prop:** Added `renderAsOverlay` prop to `WheelDatePickerModal` instead of always rendering as overlay. Preserves backward compat for onboarding screen (standalone Modal). EditProfileModal passes `renderAsOverlay` to avoid iOS stacked-modal issue.
3. **Bug 2 — Keyboard.dismiss before picker open:** Added explicit `Keyboard.dismiss()` call in date button handler to prevent Android double-tap caused by keyboard dismiss stealing first tap.
4. **Bug 5 — InteractionManager over setTimeout:** Used `InteractionManager.runAfterInteractions()` instead of a fixed delay for word input focus. More reliable across devices — waits for modal animation to complete.
